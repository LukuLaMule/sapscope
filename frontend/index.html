Comment ça marche

Opérationnel en 10 minutes.

Pas de RFC compliqués à configurer. Pas d'accès BTP. Juste un agent Python, un cron, et c'est parti.

Étape 01

Installez l'agent sur votre serveur SAP

L'agent pyrfc se déploie en quelques commandes sur n'importe quel serveur applicatif SAP. Il se connecte en RFC local pour lire les métadonnées système — sans accès externe entrant.

$curl -O https://app.sapscope.com/dist/agent.tar.gz
$tar -xzf agent.tar.gz && cd sap-agent
$pip install -r requirements.txt
$# SAPSCOPE_TOKEN=your_token_here
$python agent.py --once
✓Snapshot envoyé — PRD, 47 composants
Étape 02

Les données remontent automatiquement

Un systemd timer déclenche l'agent toutes les 24h. Chaque exécution envoie un snapshot complet : composants installés, support packages, kernel, objets custom ABAP, infos DB.

agent@PRD — actif
Dernier snapshot: il y a 2h · 47 composants · 1 247 objets custom
agent@QAS — actif
Dernier snapshot: il y a 3h · 47 composants · 1 199 objets custom
agent@DEV — inactif
Dernier snapshot: il y a 26h · Vérifier le timer
Étape 03

Consultez l'analyse IA depuis n'importe où

Claude analyse chaque snapshot et génère un rapport structuré : statut de maintenance, risques identifiés, recommandations priorisées. Accessible sur ordinateur, tablette ou mobile.
